# 代理管理模块 - 功能说明

**日期**: 2026-04-13  
**版本**: 0.3.0  
**状态**: ✅ 完全实现

---

## 📋 功能概览

代理管理模块是一个功能完整的 Mihomo/Clash 代理控制工具，提供了以下功能：

### 核心功能

1. ✅ **API 连接管理**
   - 支持手动输入 API 地址和 Secret
   - 支持从默认 profile 的 config.yaml 自动读取
   - API 地址复制到剪贴板

2. ✅ **实时监控**
   - 显示内核版本（Mihomo/Clash）
   - 显示当前运行模式（rule/global/direct）
   - 显示监听端口信息
   - 显示当前连接数

3. ✅ **代理模式切换**
   - 支持 rule、global、direct 三种模式
   - 支持切换时自动关闭连接
   - 实时更新模式显示

4. ✅ **策略组和节点管理**
   - 树形显示所有策略组和节点
   - 支持关键词筛选
   - 双击或按钮切换节点
   - 实时显示当前选中节点

5. ✅ **延迟测试**
   - 支持自定义测试 URL
   - 测试选中节点的延迟
   - 显示延迟结果

6. ✅ **连接管理**
   - 关闭全部连接
   - 实时显示连接数

7. ✅ **控制台访问**
   - 支持打开 Yacd 在线控制台
   - 支持打开本地 UI 控制台

8. ✅ **自动刷新**
   - 支持启用/禁用自动刷新
   - 可配置刷新间隔（3/5/10/15 秒）
   - 轻量刷新模式（不包含版本信息）

9. ✅ **配置持久化**
   - 自动保存 API 地址和 Secret
   - 自动保存自动刷新设置
   - 自动保存刷新间隔

---

## 🎯 详细功能说明

### 1. API 连接管理

**位置**: 第一行

**功能**:
- **API 输入框**: 输入 Mihomo/Clash 的 API 地址（默认 http://127.0.0.1:9090）
- **复制 API 按钮**: 一键复制 API 地址到剪贴板
- **Secret 输入框**: 输入 API Secret（如果需要）
- **从 profile 读取**: 勾选后自动从 ~/.linktunnel/default/config.yaml 读取 API 和 Secret

**使用场景**:
- 连接到本地 Mihomo/Clash 实例
- 连接到远程代理服务器
- 自动读取配置文件

### 2. 连接和刷新控制

**位置**: 第二行

**功能**:
- **连接/刷新按钮**: 立即连接或刷新所有数据（完整刷新）
- **自动刷新复选框**: 启用/禁用自动刷新
- **刷新间隔下拉框**: 选择自动刷新间隔（3/5/10/15 秒）

**使用场景**:
- 手动刷新代理信息
- 启用自动监控
- 调整监控频率

### 3. 实时状态显示

**位置**: 第三行

**显示内容**:
- **内核**: 显示 Mihomo 或 Clash 的版本号
- **模式**: 显示当前运行模式（rule/global/direct）
- **监听**: 显示监听的端口信息（HTTP、SOCKS5 等）
- **连接数**: 显示当前活跃连接数

**更新频率**:
- 手动刷新时立即更新
- 自动刷新时定期更新

### 4. 代理模式切换

**位置**: 第三行

**功能**:
- **模式下拉框**: 选择要切换的模式（rule/global/direct）
- **应用模式按钮**: 应用选中的模式
- **切换时关闭连接**: 勾选后切换模式时自动关闭所有连接

**模式说明**:
- **rule**: 规则模式，根据规则决定是否走代理
- **global**: 全局模式，所有流量都走代理
- **direct**: 直连模式，不走代理

### 5. 策略组和节点管理

**位置**: 树形列表区域

**功能**:
- **树形显示**: 显示所有策略组及其包含的节点
- **当前节点**: 显示每个策略组当前选中的节点
- **双击切换**: 双击节点可直接切换
- **按钮切换**: 选中节点后点击"应用节点"按钮切换

**使用方法**:
1. 在树形列表中找到要切换的节点
2. 双击节点或选中后点击"应用节点"按钮
3. 等待切换完成

### 6. 筛选功能

**位置**: 第五行

**功能**:
- **筛选输入框**: 输入关键词筛选节点
- **实时筛选**: 输入时自动筛选匹配的节点
- **支持策略组和节点名称筛选**

**使用方法**:
1. 在筛选框输入关键词（如节点名称的一部分）
2. 树形列表自动显示匹配的节点
3. 清空筛选框显示所有节点

### 7. 延迟测试

**位置**: 第五行

**功能**:
- **测试 URL 输入框**: 输入延迟测试 URL（默认 http://www.gstatic.com/generate_204）
- **测延迟按钮**: 测试选中节点的延迟
- **延迟结果**: 显示测试结果（毫秒）

**使用方法**:
1. 在树形列表中选中一个节点
2. 输入测试 URL（可选，使用默认值）
3. 点击"测延迟（选中节点）"按钮
4. 等待测试完成，查看结果

### 8. 连接管理

**位置**: 第五行

**功能**:
- **关闭全部连接按钮**: 一键关闭所有活跃连接
- **连接数显示**: 实时显示当前连接数

**使用场景**:
- 清理僵尸连接
- 重新建立连接
- 测试连接稳定性

### 9. 控制台访问

**位置**: 第五行

**功能**:
- **打开控制台按钮**: 打开 Mihomo/Clash 的 Web 控制台
- **支持两种控制台**:
  - Yacd: 在线控制台（功能完整）
  - 本地 UI: 本地嵌入式控制台

**使用方法**:
1. 点击"打开控制台"按钮
2. 选择要打开的控制台类型
3. 浏览器自动打开控制台

### 10. 快捷键

**支持的快捷键**:
- **F5**: 全量刷新（重新获取所有数据）
- **双击节点**: 切换节点

## 🔧 技术实现

### 并行请求

代理管理模块使用 ThreadPoolExecutor 并行请求多个 API 端点，提高响应速度：

```python
# 完整刷新：并行请求 4 个端点
- /version (内核版本)
- /configs (配置信息)
- /proxies (策略组和节点)
- /connections (连接信息)

# 轻量刷新：并行请求 3 个端点
- /configs (配置信息)
- /proxies (策略组和节点)
- /connections (连接信息)
```

### 异步处理

所有网络请求都在后台线程中执行，不阻塞 UI：

```python
def _on_refresh(self, *, full: bool = True) -> None:
    # 在后台线程中执行网络请求
    threading.Thread(target=job, daemon=True).start()
```

### 队列通信

使用 queue.Queue 在线程间安全地传递数据：

```python
# 后台线程将结果放入队列
self._result_queue.put(("ok", snap, "已更新"))

# 主线程定期轮询队列
self._poll_timer.timeout.connect(self._poll_queue)
```

### 防抖处理

筛选功能使用防抖处理，避免频繁重建树形列表：

```python
# 200ms 防抖
self._filter_timer = QTimer(self)
self._filter_timer.setSingleShot(True)
self._filter_timer.timeout.connect(self._rebuild_tree)
self._filter_timer.start(200)
```

## 📊 API 端点说明

### 支持的 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| /version | GET | 获取内核版本 |
| /configs | GET/PATCH | 获取/修改配置 |
| /proxies | GET/PUT | 获取/切换代理 |
| /connections | GET/DELETE | 获取/关闭连接 |

### 认证

所有请求都支持 Secret 认证：

```python
# 在请求头中添加 Authorization
headers = {"Authorization": f"Bearer {secret}"}
```

## 🚀 使用示例

### 示例 1: 连接到本地 Mihomo

1. 启动 Mihomo: `mihomo -d ~/.linktunnel/default`
2. 打开代理管理模块
3. 确保 API 地址为 `http://127.0.0.1:9090`
4. 点击"连接/刷新"按钮
5. 查看代理信息和节点列表

### 示例 2: 切换代理节点

1. 在树形列表中找到要切换的节点
2. 双击节点或选中后点击"应用节点"
3. 等待切换完成
4. 查看"当前"显示已更新

### 示例 3: 测试节点延迟

1. 在树形列表中选中一个节点
2. 点击"测延迟（选中节点）"按钮
3. 等待测试完成
4. 在日志中查看延迟结果

### 示例 4: 启用自动刷新

1. 勾选"自动刷新"复选框
2. 选择刷新间隔（如 5 秒）
3. 模块将每 5 秒自动刷新一次数据
4. 取消勾选以停止自动刷新

## ⚙️ 配置说明

### 保存的配置项

模块会自动保存以下配置：

```python
{
    "api": "http://127.0.0.1:9090",      # API 地址
    "secret": "",                         # API Secret
    "from_profile": True,                 # 是否从 profile 读取
    "auto_refresh": False,                # 是否启用自动刷新
    "auto_refresh_interval": 5            # 自动刷新间隔（秒）
}
```

### 配置文件位置

配置保存在 `~/.linktunnel/config.json` 中的 `proxy` 部分

## 🐛 故障排除

### 问题 1: 无法连接到 API

**原因**: API 地址错误或 Mihomo/Clash 未运行

**解决方案**:
1. 检查 API 地址是否正确
2. 确保 Mihomo/Clash 已启动
3. 检查防火墙设置

### 问题 2: 显示"API 错误"

**原因**: API 请求失败

**解决方案**:
1. 检查网络连接
2. 检查 Secret 是否正确
3. 查看日志了解详细错误信息

### 问题 3: 节点列表为空

**原因**: 代理配置中没有策略组

**解决方案**:
1. 检查 Mihomo/Clash 配置文件
2. 确保配置中有策略组定义
3. 重新启动 Mihomo/Clash
